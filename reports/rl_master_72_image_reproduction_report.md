# 72 张图 RL-master FastApply GRPO 迁移复现报告
本报告基于 `/root/rlhf/images` 的 72 张参考图，把图片中的 RL-master / verl 风格 FastApply GRPO pipeline 迁移到当前项目。这里不是 SFT；训练主线是 GRPO：同一 prompt 采样多条 response，计算 reward，做 group-relative advantage，再进入 policy/KL/entropy loss 更新。
## 已迁移的源码结构
- `rl_master/configs/config-rl-fastapply.yaml`：rollout n=8、seq_len=8192、temperature/top_p=0.99、top_k=50、GRPO advantage 等配置。
- `rl_master/scripts/run_fastapply_grpo_smoke.py`：离线可跑的完整 GRPO smoke，复刻 `data -> rollout -> reward -> advantage -> loss -> report`。
- `rl_master/scripts/run_fastapply_ver1.py`：FastApply 环境和 rule reward smoke。
- `rl_master/applications/fastapply/reward/{score.py,utils.py}`：rule exact compare、diff view、judge prompt fallback。
- `rl_master/agent/environment/packages/fastapply/impl.py`：FastApply 环境，reward = `0.8 * correctness + 0.2 * validity`。
- `rl_master/prompts/{apply.yaml,generate.yaml,judge.yaml}`：图片里的 apply/generate/judge prompt 协议（实际文件名为 `judge.yaml`）。
- `rl_master/agent/evaluation/*`：extractor、strategy、agent、pipeline、evalfastapply。
- `rl_master/agent/common/models/*`：LLM、Inference、PromptTemplate、JSON/pattern parser。
- `fastapply_pipeline/train_mint_fastapply_grpo.py`：真实 MinT Qwen3 4B LoRA GRPO 入口，使用 `loss_fn="importance_sampling"`。

## 可运行验证
```bash
./mint_simple_training/.venv/bin/python -m rl_master.scripts.run_fastapply_ver1
./mint_simple_training/.venv/bin/python -m rl_master.scripts.run_fastapply_grpo_smoke
./mint_simple_training/.venv/bin/python -m rl_master.scripts.evaluation.evalfastapply
./mint_simple_training/.venv/bin/python tests/test_rl_master_fastapply.py
```
已在服务器跑通：`run_fastapply_ver1` reward=1.0；`run_fastapply_grpo_smoke` 生成 8 条 rollout、mean_reward/mean_loss 和逐样本 loss；`evalfastapply` accuracy=1.0；本地断言测试通过。输出报告见 `reports/rl_master_fastapply_smoke.json` 与 `reports/rl_master_fastapply_grpo_smoke.json`。

## 与图片 pipeline 的关键一致点
1. **GRPO 而不是 SFT**：同组采样 `n=8`，每条 response 独立 reward，advantage 在组内中心化/标准化。
2. **Reward 两段式**：先 rule exact compare（忽略空白行），失败再进入 judge prompt；环境 reward 按 `0.8 correctness + 0.2 validity` 聚合。
3. **Loss 结构**：policy surrogate 使用 old/current logprob ratio 和 advantage；同时保留 ref policy 的 KL 惩罚与 entropy bonus。真实 MinT 训练入口用 `forward_backward(datums, loss_fn="importance_sampling")`。
4. **LoRA 位置**：真实 MinT 入口创建 `create_lora_training_client(... rank=16, train_mlp=True, train_attn=True, train_unembed=True)`，对应插在 Qwen3 4B transformer 的 attention、MLP 和 unembed 相关线性层上。
5. **数据策略**：当前只放 1-2 条 tiny test 数据用于能跑通，不虚构大数据集；后续可把真实 FastApply 样本喂给同样入口。
